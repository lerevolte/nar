## Деплой (на сервере)

```bash
cd /var/www/narepite-web
git pull
```

## ТЗ 9. Оценка оптимизации мобильной скорости (обратная связь)

Главные виновники низких баллов (39 на главной):

### Быстро и дёшево (уже сделано в этом ПР)
1. **Кэш-бастер `?v=time()`** — статика (CSS/JS) сбрасывала кэш браузера каждую секунду, т.е. кэширования не было вообще. Заменено на версию по mtime файла. Эффект: повторные визиты и переходы между страницами заметно быстрее. Срок: сделано.
2. **loading="lazy" + width/height** для изображений ниже первого экрана на главной (обложки треков, иконки преимуществ). Сделано.

### Быстро, но требует доступа к серверу (1-2 часа)
3. **Cache-Control для статики в nginx** — добавить в конфиг:
   ```nginx
   location ~* \.(css|js|svg|png|jpg|jpeg|webp|avif|woff2?|ico)$ {
       expires 30d;
       add_header Cache-Control "public, max-age=2592000, immutable";
   }
   ```
   Приоритет: высокий, риск: нулевой.
4. **gzip/brotli** — проверить `gzip on` для text/css, application/javascript. Приоритет: высокий.

### Средняя сложность (0.5–1 день)
5. **Tailwind CDN (`cdn.tailwindcss.com`)** — это dev-инструмент: ~300+ КБ JS, компилирующий стили в рантайме. Главный источник TBT и блокировки рендера. Правильное решение — собрать Tailwind через Vite (vite + laravel-vite-plugin уже в проекте) и подключать статический CSS. Это самый большой резерв по баллам PSI. Приоритет: высокий.
6. **Fancybox с CDN (jsdelivr)** — перенести локально/в сборку, подключать defer и только на страницах, где есть галереи. Приоритет: средний.
7. **Яндекс.Метрика** — грузить после onload/через 3 сек. Экономит TBT на мобильных. Приоритет: средний.
8. **Обложки треков (main page)** — конвертация в WebP при сохранении + `srcset`. Приоритет: средний (~78 КБ по отчёту PSI).
9. **preload LCP-элемента** — после п.5 LCP скорее всего станет текстовым блоком; оценить повторно по новому отчёту PSI.

### Требует доработки архитектуры (оценивать отдельно)
10. **TTFB** — замерить отдельно: включить opcache (если не включён), кэш ответов лендинга (`Cache::remember` для topTracks), проверить N+1. Оценка после профилирования.
11. **Неиспользуемый JS (~125 КБ)** — большая часть уйдёт вместе с Tailwind CDN (п.5). Остальное — ревизия по страницам.

**Рекомендуемый порядок:** п.3–4 (nginx) → п.5 (Tailwind build) → повторный замер PSI → решение по остальному.
**Реалистичная цель первой итерации:** главная с 39 до ~60–70, LCP < 5 с, TBT < 400 мс.
